# Changelog

## 0.1.2-beta

Релизный аудит качества перед публичным использованием.

### Исправлено
- Chromium: диагностика теперь сначала выполняет реальный headless-запуск браузера и только после отказа анализирует stderr
- Chromium: поиск CloakBrowser-бинарника учитывает `CLOAKBROWSER_BINARY_PATH` и кэш `~/.cloakbrowser/chromium-*`
- Chromium: Debian 12 больше не получает рекомендации установить `t64`-пакеты
- Chromium: Arch больше не показывает команды установки для уже установленных пакетов
- Chromium: добавлены Void, Gentoo и отдельная карта openSUSE
- Установщик: `bun install` больше не маскирует ошибки через pipeline
- Установщик: `qf` проверяется по точному пути, а не по любому бинарнику в PATH
- Установщик: PATH добавляется в начало, wrapper указывает на фактический `INSTALL_DIR`
- Установщик: запуск от root блокируется по умолчанию, чтобы не устанавливать проект в `/root`
- HTTP API: убран открытый CORS `*`, добавлена проверка Origin для localhost
- HTTP API: добавлена валидация JSON, email/password и защита от параллельных операций браузера
- Безопасность: activation link теперь разрешён только для `https://*.qwen.ai`
- Безопасность: пароли и email генерируются через криптографический RNG
- Хранилище: `accounts.json` пишется с правами `0600`, очередь записи восстанавливается после ошибки
- Runtime: lock-файл создаётся атомарно через `wx`
- Runtime: browser start защищён от race condition и утечки второго контекста
- Runtime: crash handlers выполняют полный shutdown, а не только удаление lock-файла

### Документация
- README и README.en обновлены под фактический алгоритм диагностики Chromium
- Документация установки уточняет user-local установку, PATH и поддерживаемые Linux-дистрибутивы
- Версия обновлена до 0.1.2-beta

## 0.1.1-beta

Исправление критических ошибок после первой публичной беты.

### Исправлено
- Установщик: `command -v qf` теперь реально проверяет PATH, никогда не показывает ложное сообщение
- Установщик: корректный экспорт PATH с `hash -r` перед проверкой
- Первый запуск: мастер языка теперь выполняется до блокировки, чтобы гарантировать открытие меню
- Первый запуск: ошибка HTTP-сервера больше не блокирует запуск
- Chromium: отсутствующие библиотеки теперь показывают понятную ошибку с командой установки
- Chromium: единый модуль диагностики используется в install.sh, doctor, bootstrap и перед регистрацией
- Chromium: реальный тест запуска бинарника (не только проверка .so файлов)
- Документация: добавлен раздел «Требования к Chromium»
- Docker: проверена работа на Debian Bookworm
- Идемпотентность: повторный запуск install.sh безопасен, без дублирования PATH

### Изменения
- Версия обновлена до 0.1.1-beta

## 0.1.0-beta

Первая публичная бета-версия.

### Добавлено
- Автоматическая регистрация аккаунтов Qwen
- Подтверждение email через CatchMail API
- Автоматическая активация по ссылке из письма
- Пакетное создание (до 50 аккаунтов)
- HTTP API для интеграции с внешними скриптами
- Диагностика системы
- Статистика (сессия + история)
- Два языка интерфейса (RU/EN)
- Graceful shutdown

### Архитектура
- `BrowserManager` — единственный владелец браузера
- `context.ts` — разрыв циклической зависимости AppContext
- logout в `finally` — гарантированная очистка после любой операции
- `waitUntil: 'domcontentloaded'` — предотвращение зависаний на WebSocket

### Установка
- `install.sh` — curl-to-bash инсталлятор
- `bin/qf` — глобальная команда
