# Changelog

## 0.1.1-beta

Исправление критических ошибок после первой публичной беты.

### Исправлено
- Установщик: `command -v qf` теперь реально проверяет PATH, никогда не показывает ложное сообщение
- Установщик: корректный экспорт PATH с `hash -r` перед проверкой
- Первый запуск: мастер языка теперь выполняется до блокировки, чтобы гарантировать открытие меню
- Первый запуск: ошибка HTTP-сервера больше не блокирует запуск
- Chromium: отсутствующие библиотеки теперь показывают понятную ошибку с командой установки
- Chromium: единый модуль диагностики используется в install.sh, doctor, bootstrap и перед регистрацией
- Chromium: реальный тест запуска бинарника (не только проверка .so файлов)
- Документация: добавлен раздел «Требования к Chromium»
- Docker: проверена работа на Debian Bookworm
- Идемпотентность: повторный запуск install.sh безопасен, без дублирования PATH

### Изменения
- Версия обновлена до 0.1.1-beta

## 0.1.0-beta

Первая публичная бета-версия.

### Добавлено
- Автоматическая регистрация аккаунтов Qwen
- Подтверждение email через CatchMail API
- Автоматическая активация по ссылке из письма
- Пакетное создание (до 50 аккаунтов)
- HTTP API для интеграции с внешними скриптами
- Диагностика системы
- Статистика (сессия + история)
- Два языка интерфейса (RU/EN)
- Graceful shutdown

### Архитектура
- `BrowserManager` — единственный владелец браузера
- `context.ts` — разрыв циклической зависимости AppContext
- logout в `finally` — гарантированная очистка после любой операции
- `waitUntil: 'domcontentloaded'` — предотвращение зависаний на WebSocket

### Установка
- `install.sh` — curl-to-bash инсталлятор
- `bin/qf` — глобальная команда
